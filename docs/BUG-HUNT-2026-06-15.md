# Bug hunt + test-hardening pass — 2026-06-15

Goal (from Kirill): find, reproduce-with-tests, and fix as many bugs as
possible — soundness in storage, spec/cornercase gaps, whole *classes* of
bugs — and generally raise test-suite quality, robustness, and coverage.
Refactoring for testability is allowed **only** if behaviour is preserved;
the project is in a carefully hand-tuned state and hard-won microbehaviours
must not be sacrificed. Shield against accidental breakage:
**characterization tests** — pin current observable behaviour first, then
any change that alters it shows up red.

## Method

1. Read every core module + the app's pure-logic/test surfaces.
2. Verify the scariest soundness hypotheses *empirically* before touching
   code (a "fix" to a non-bug is exactly how niceties die).
3. Multi-agent fan-out hunt + adversarial verification (real bug vs.
   deliberate nicety) — workflow.
4. Implement confirmed fixes + regression tests serially in the main repo,
   running the full suite after each change.

## Method, part 2: the fan-out

A 2-phase workflow ran: 6 module-cluster hunters + 1 tooling researcher
(read-only, adversarial), then a skeptic verified **every** candidate as
*real bug* vs *not-a-bug* vs *deliberate nicety* vs *needs-human-judgment*,
each with a behaviour-preserving fix, a regression test, and explicit
"microbehaviour this must NOT disturb" notes. 15 candidates → **10 real
bugs, 3 not-a-bug, 2 design calls**. The verifiers actually compiled and
reverted their checks (hence the transient stray-file diagnostics during the
run; the tree was clean afterward).

The behaviour-preservation shield (Kirill's ask: don't sacrifice hand-tuned
microbehaviours): every fix ships with a test that pins the *new* correct
behaviour AND a guard asserting the neighbouring behaviour is unchanged; the
two genuinely ambiguous calls were **left as-is** and handed back, not
"fixed".

## Fixed (10) — each with a regression test

All char-coords/byte-coords claims re-verified against source before editing.

1. **llm `parse_chat_response` panic on a multibyte body** (`llm.rs`).
   `&text[..text.len().min(200)]` panics when byte 200 is mid-char (garbage
   non-JSON reply ≥200 bytes). Fix: char-boundary-safe `body_preview`.
   *(The one hypothesis I'd pre-confirmed.)*
2. **Adjacent fenced code blocks fuse on `.md` import** (`markdown.rs`,
   `from_markdown`). Two `` ``` `` fences with a blank line between collapsed
   into one block, dropping the boundary newline (`foo\nbar` → `foobar`).
   Fix: a `code_fresh` flag forces a new block at each fence's first line
   while preserving single-fence multi-`Text`-event continuation.
3. **Repeated diagnosis quotes all re-anchor to the first occurrence**
   (`diagnose.rs`, `to_annotations`). `cursor = range.start` never advanced
   past a match. Fix: `cursor = range.end` (matches the contract the
   `anchor` doctest already demonstrates; the `or_else` wrap-around keeps
   single/out-of-order quotes working).
4. **BlockMap invariant broke on non-LF Unicode separators** (`document.rs`).
   `merged`/`splits` counted only `'\n'`; a paste of CR / VT / FF / NEL /
   U+2028 / U+2029 (classic-Mac, PDF copy) desynced `kinds.len()` from
   `rope.len_lines()`. Fix: count breaks via ropey's own metric
   (`char_to_line` delta / `Rope::from_str(text).len_lines()-1`) — CRLF
   correctly counts as one. `'\n'`-only typing is byte-identical.
5. **Newline in a block-kind string field collapsed the whole BlockMap on
   reopen** (`store.rs`). Block kinds were newline-joined tokens; a `'\n'`
   in `CodeBlock.info` / `Image.src` (reachable via `.md` import of an
   entity-encoded URL) became a token boundary, desyncing the count →
   `Document::new` reset every block to Paragraph on the *second* open. Fix:
   persist kinds as **serde_json** (like history/annotations already are),
   read JSON-first with a legacy-token fallback so old `.strop` files and
   older Loro frontiers (via `state_at`) still load.
6. **Image alt/caption silently dropped on `.strop` save/reopen**
   (`store.rs`). Same root cause as #5; the JSON fix carries alt/caption
   that the token format discarded (a shipped alt-text editor writes them).
7. **Image alt/src emitted unescaped in `to_markdown`** (`markdown.rs`). An
   alt containing `]` (author-editable) or a src with spaces/parens broke
   the `![…](…)` on markdown round-trip. Fix: escape alt via `escape_into`,
   angle-bracket such srcs. Plain alt + asset/relative src export
   byte-identically (guards the editor's `](asset:…)` rewrite).
8. **Trailing U+2028 hard break exported a stray backslash** (`markdown.rs`).
   A block-final hard break is unrepresentable in CommonMark; export emitted
   `\\\n` that re-imported as a literal `\`. Fix: drop a block-trailing
   U+2028 run; mid-block breaks still export normally.
9. **`list_models` hid auth/provider failures as an empty list** (`llm.rs`).
   `http_status_as_error(false)` makes 401/429/5xx arrive as `Ok`; the
   parser mapped a `{"error":…}` body to `data: []`. Fix: factor
   `parse_models_response(status, body)` routing ≥400 to the same error
   kinds as `chat()`; a genuine 200-empty still returns `Ok(vec![])`.
10. **Word-motion stack overflow on a long blank-line run** (`editor.rs`).
    `previous/next_word_boundary` recursed one frame per consecutive empty
    line → uncatchable abort on a tens-of-thousands-blank-line paste. Fix:
    extracted iterative free functions over `&Document` (byte-identical,
    now unit-testable).
11. **NoteInput stale `marked` after backspace → IME mid-char panic**
    (`editor.rs`). A manual backspace during an active IME composition left
    a byte-length `marked` that the next preedit truncation split mid-char.
    Fix: clear `marked` on backspace/backspace_word (no-op when not
    composing) + a char-boundary-safe `drop_marked_tail` helper. Reachable
    on macOS-style dispatch; effectively unreachable on the dev Wayland/IME
    setup, but the fix is a strict no-op on the common path.

## Left as-is — 2 design calls handed back (NOT bugs to silently change)

- **Atomic save uses a fixed `*.strop.tmp` name** (`store.rs:save`). Two
  *processes* saving the same file can tear the temp. But the app is
  explicitly "one document, one window, one process" (editor.rs comment); a
  pid/nonce temp name does **not** make multi-process safe (still
  last-writer-wins) and would leak a *visible* `Doc.strop.<pid>.tmp` in the
  user's Documents folder on a hard crash, whereas the fixed name
  self-cleans. Genuine tradeoff in an unsupported config → your call. A
  hardened version (per-writer temp, same-dir, sweep stale siblings on save)
  is sketched in the workflow output if you ever want multi-process safety.
- **Checkpoint restore collapses note anchors to a point at doc end**
  (`document.rs:restore_state`). This *follows the documented "fully-deleted
  anchor orphans" rule* (a wholesale restore deletes all text), bodies/
  statuses/count all survive, and undo restores the exact overlay. "Fixing"
  it to keep clamped offsets is a **semantics choice** (old offsets live in
  the old text's coordinate space) — exactly the kind of hand-tuned call I
  won't make unilaterally. A `clamp_to` variant is ready if you decide notes
  should keep positions across a restore.

## Test suite hardening (new)

- `crates/strop-core/tests/robustness.rs` (9 properties):
  - **No untrusted input panics** — `from_markdown`, `typograph::process`,
    `extract_json_array`, `diagnose::anchor`, `import_image`, and
    `Store::open` over arbitrary (multibyte) input / arbitrary bytes. Locks
    the char/byte-boundary class shut.
  - **Full save→reopen state round-trip** (the biggest gap model.rs left):
    a random op sequence mirrored into the store as the editor does, then
    `save_with_state` → `Store::open`, asserting text + span coverage +
    block kinds + notes all survive. The generator deliberately emits Image
    kinds with `]`-bearing alt and newline-bearing caption (exercises #5/#6).
  - **SpanSet invariants** (sorted, non-empty, self-covering).
  - **diff block reconstruction byte-exact** over arbitrary paragraph lists.
- Per-fix regression tests added inline (`llm`, `diagnose`, `document`,
  `markdown`, `store`, `editor`).
- Validated at `PROPTEST_CASES=4096` (model.rs + robustness): green, ~1.5s.
- `.gitignore` only excludes `/target`, so any future proptest
  counterexample seed under `proptest-regressions/` is committed
  automatically — no change needed.

Tally: **103 → 125 tests**, all green; `cargo clippy --workspace
--all-targets` clean.

## Tooling research (Kirill's explicit ask) — priority order

1. **`cargo-llvm-cov`** (low effort, high fit) — run once to *map* untested
   branches (esp. the 10k-line editor.rs coordinate-mapping surface and core
   error arms). It's the map; it catches nothing itself.
2. **The cheap no-panic + save→reopen proptests** — done above (no nightly).
   The persistence loop was the single biggest hole.
3. **Commit `proptest-regressions/` + a high-`PROPTEST_CASES` CI lane** — the
   harness already reads `PROPTEST_CASES`; this is config-only. (No CI exists
   yet — `.github/workflows` is absent. A workflow running `cargo test` at
   elevated cases + `cargo llvm-cov` + periodic `cargo mutants -p
   strop-core` would make all of this durable. Left for you to opt into —
   it's outward-facing infra.)
4. **`cargo-mutants -p strop-core`** (medium) — the codebase has rich
   invariant tests, so the real risk is assertions that look strong but
   don't pin the exact boundary (the off-by-one class). Run with
   `PROPTEST_CASES=32` + a generous timeout. **Do NOT** point it at
   strop-app (gpui rebuilds are prohibitive).
5. **`cargo-fuzz` + structure-aware corpora** (medium) — for `Store::open`
   (mutate real `.strop` snapshots; random bytes only hit Loro's clean Err
   path) and `import_image` (mutate real PNG/JPEG/GIF/TIFF). Needs nightly +
   a `fuzz/` crate.
- **Skip miri** (core has zero `unsafe`; the char-boundary class is *safe*
  panics that ordinary tests already surface; loro/image/blake3 are
  impractical under miri) and **skip loom** (no hand-rolled shared-memory
  concurrency — gpui is single-threaded entity + message passing).
